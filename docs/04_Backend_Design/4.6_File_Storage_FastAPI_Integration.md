# Purpose

This document defines how ExperiML stores datasets and machine learning artifacts, and how the Express backend collaborates with the FastAPI service to execute machine learning workflows.

The architecture separates application management from machine learning execution, allowing each service to focus on its respective responsibilities while remaining independently maintainable and scalable.

---

# Architecture Overview

ExperiML follows a service-oriented architecture consisting of two backend services.

- **Express Backend** manages authentication, authorization, API requests, database operations, business logic, and workflow orchestration.
- **FastAPI Service** performs computationally intensive machine learning tasks such as data preprocessing, model training, evaluation, and prediction.

Uploaded datasets and generated artifacts are stored on the file system, while their metadata is maintained within the database.

---

# File Storage Strategy

Large files are stored on the local file system instead of the relational database.

This approach improves performance, simplifies file management, and prevents unnecessary database growth.

Recommended storage layout:

```text
storage/
│
├── datasets/
│   └── {projectId}/
│       └── dataset.csv
│
├── artifacts/
│   └── {experimentId}/
│       ├── model.pkl
│       ├── metrics.json
│       ├── confusion_matrix.png
│       └── feature_importance.png
│
└── predictions/
    └── {experimentId}/
        └── prediction_results.csv
```

Database records reference these files through metadata, enabling efficient searching and retrieval without storing the files themselves.

---

# Storage Abstraction

ExperiML accesses file storage through a dedicated **Storage Service** and **Storage Client**.

```text
Business Module
      │
      ▼
Storage Service
      │
      ▼
Storage Client
      │
      ▼
Local File System
```

This abstraction separates business logic from storage implementation.

The initial implementation uses the local file system. Future versions can migrate to cloud object storage (such as AWS S3, Azure Blob Storage, or Google Cloud Storage) by replacing the Storage Client without affecting business modules.

# FastAPI Integration

The Express backend and FastAPI service have clearly separated responsibilities.

| Express Backend | FastAPI Service |
|-----------------|-----------------|
| Authentication | Data preprocessing |
| Authorization | Feature engineering |
| Request validation | Model training |
| Business logic | Model evaluation |
| Database operations | Prediction |
| File management | Artifact generation |
| Workflow orchestration | Machine learning execution |

Express remains the primary entry point for all client requests, while FastAPI acts as a specialized machine learning service.

# Machine Learning Workflow

A typical machine learning workflow follows these steps.

```text
Dataset Upload
      │
      ▼
Dataset Stored
      │
      ▼
Experiment Created
      │
      ▼
FastAPI Training
      │
      ▼
Artifacts Generated
      │
      ▼
Metadata Stored
      │
      ▼
Results Returned
```

This workflow ensures that machine learning execution remains independent from application management.

# Artifact Management

Machine learning experiments generate several output artifacts.

Common artifact types include:

- Trained model (`.pkl`)
- Evaluation metrics (`.json`)
- Confusion matrix (`.png`)
- ROC curve (`.png`)
- Feature importance visualization (`.png`)
- Prediction results (`.csv`)

Artifacts are stored on the file system, while metadata such as file name, type, size, storage path, creation time, and associated experiment is maintained in the database.

This approach enables efficient organization, retrieval, and lifecycle management of generated files.

---

# Design Decisions

The storage and integration architecture follows these principles:

- File system storage for large datasets and generated artifacts.
- Database stores metadata rather than binary files.
- Storage abstraction through dedicated Service and Client layers.
- Express orchestrates application workflows.
- FastAPI performs machine learning execution.
- Clear separation between application logic and computational processing.
- Storage architecture designed for future cloud migration.

---

# Summary

ExperiML separates application management, machine learning execution, and file storage into distinct architectural responsibilities.

The Express backend orchestrates workflows, the FastAPI service executes machine learning tasks, and the storage layer manages datasets and generated artifacts through an abstraction that supports future migration to cloud storage. This design promotes maintainability, scalability, and clean separation of concerns while providing an efficient foundation for machine learning experimentation.
