Infrastructure components are separated into **Clients** and **Services**.

---

# Design Decisions

The backend folder structure follows these architectural decisions:

- Feature-based module organization.
- Consistent internal structure across all modules.
- Separation of Domain Modules and Infrastructure.
- External systems accessed through Infrastructure Clients.
- Infrastructure Services provide application-level abstractions.
- Shared resources centralized for reuse.
- Scalable directory layout supporting future modules and integrations.

---

# Summary

The ExperiML backend uses a modular, feature-based directory structure that separates business logic, infrastructure integrations, shared utilities, and configuration.

This organization improves maintainability, encourages consistent development practices, and provides a scalable foundation for future enhancements while keeping responsibilities clearly separated.
