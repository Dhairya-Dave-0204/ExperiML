# Purpose

This document defines the visual identity and design system used throughout the ExperiML web application.

It establishes a consistent set of design principles, colors, typography, spacing, layout rules, animations, icons, and responsive behavior to ensure a cohesive user experience across the platform.

---

# Design Philosophy

ExperiML follows a modern enterprise SaaS design language.

The interface prioritizes clarity, usability, accessibility, and consistency over decorative visual effects.

Design goals include:

- Professional appearance
- Clean visual hierarchy
- Minimal visual noise
- Consistent spacing
- High readability
- Responsive layouts
- Accessible interactions
- Purposeful animations

The interface intentionally avoids excessive gradients, glassmorphism, neon colors, oversized shadows, and unnecessary animations commonly found in template-driven or "vibe-coded" applications.

| Property | Value |
|----------|-------|
| Maximum Content Width | 1440px |
| Grid | 12 Columns |
| Container Padding | 24px |

Section spacing:

| Device | Vertical Padding |
|---------|------------------|

ExperiML uses **Lucide React** as the primary icon library.

- Consistent stroke-based design
- Lightweight and tree-shakeable

- Extensive icon collection
- Suitable for dashboards and enterprise applications
- Seamless compatibility with Tailwind CSS

- 18px
- 20px
- 24px

Stroke width:

- 1.75–2

# Motion Design

Animations are implemented using **Framer Motion**.

Motion should improve usability rather than distract users.

Recommended duration:

- Fast: 150ms
- Standard: 200–250ms
- Slow: 300ms

Recommended easing:

- easeOut

Preferred animated properties:

- opacity
- transform
- scale

Avoid animating:

- width
- height
- margin
- padding
- layout properties

Common animation patterns include:

- Page transitions
- Fade-in sections
- Card hover effects
- Modal animations
- Drawer transitions
- Accordion expansion
- Skeleton loading
- Progress indicators
- Toast notifications

Continuous decorative animations should be avoided.

Reduced motion preferences should be respected.

# Accessibility

The interface follows accessibility best practices.

Key principles include:

- Sufficient color contrast
- Keyboard navigation
- Visible focus states
- Semantic HTML
- Accessible form labels
- Responsive text sizing
- Meaningful icon usage
- Reduced motion support

---

# Tailwind CSS Design Tokens

The Tailwind configuration should expose the design system through reusable theme tokens.

Recommended extensions include:

- Custom color palette
- Font families
- Border radius
- Shadows
- Container sizes
- Spacing scale
- Animation durations
- Z-index scale

These tokens become the single source of truth for all UI components.

# Summary

The ExperiML Design System establishes a unified visual language that combines modern enterprise aesthetics with responsive layouts, accessible interactions, and purposeful motion.

By standardizing colors, typography, spacing, components, animations, and design tokens, the application delivers a professional, maintainable, and consistent user experience across all pages and devices.
