# Purpose

This document defines how ExperiML manages files throughout their lifecycle, from upload to deletion.

The application manages datasets, machine learning artifacts, visualizations, and prediction outputs while ensuring secure storage, organized access, and consistent lifecycle management.

---

# Overview

ExperiML handles multiple categories of files that are generated or consumed during machine learning workflows.

These files are stored separately from the relational database, while their metadata is maintained within the database for efficient organization, searching, and retrieval.

File management is designed to provide:

- Secure file uploads
- Organized storage
- Controlled access
- Artifact lifecycle management
- Consistent cleanup procedures

---

# File Categories

| Category | Examples |

Each file category follows a consistent lifecycle regardless of its purpose within the application.

# File Lifecycle

The following lifecycle is followed for all managed files.

```text
                 Upload
                    │
                    ▼
              File Validation
                    │
                    ▼
           Metadata Generation
                    │
                    ▼
             Secure Storage
                    │
                    ▼
        ML Processing (if applicable)
                    │
                    ▼
          Artifact Generation
                    │
                    ▼
         View / Download Access
                    │
                    ▼
          Archive or Deletion
```

This lifecycle ensures that every file is validated, securely stored, tracked through metadata, and managed until its removal.

---

# Upload Management

File uploads follow a controlled process before being accepted into the system.

The upload process includes:

- File type validation
- File size validation
- Secure file naming
- Metadata creation
- Storage location assignment

Only supported file formats are accepted, reducing the risk of invalid or malicious uploads.

---

# Storage Organization

ExperiML stores uploaded files on the local file system while maintaining metadata in the database.

Storage is organized using the project and experiment hierarchy to simplify management and retrieval.

Typical organization includes:

- Project datasets
- Experiment artifacts
- Prediction outputs

The storage architecture itself is documented in **Chapter 4.6 – File Storage & FastAPI Integration**.

---

# Download Management

File downloads are performed through authenticated API endpoints.

Before a file is served, the backend verifies:

- User authentication
- Resource ownership
- File existence
- Metadata consistency

Only authorized users can access files associated with their own projects and experiments.

---

# File Deletion

File deletion follows a controlled lifecycle to maintain storage consistency.

Deletion operations include:

- Removing the physical file from storage
- Updating or removing associated metadata
- Cleaning up experiment-related artifacts when required

The application ensures that orphaned files and inconsistent metadata are avoided.

---

# Security Considerations

ExperiML applies several security practices to file management.

These include:

- Validation of uploaded file types
- File size restrictions
- Secure storage locations
- Authenticated download endpoints
- Resource ownership verification
- Controlled deletion operations

These measures help protect the integrity and confidentiality of stored files.

---

# Design Decisions

The file management architecture follows these principles:

- Files stored outside the relational database.
- Metadata maintained within the database.
- Secure upload validation.
- Organized storage hierarchy.
- Authenticated file access.
- Controlled lifecycle management.
- Extensible storage architecture.

---

# Summary

ExperiML provides a structured file management system that governs the complete lifecycle of datasets, machine learning artifacts, visualizations, and prediction outputs.

By separating file storage from metadata, validating uploads, securing downloads, and maintaining a consistent lifecycle, the application ensures that files remain organized, secure, and easy to manage throughout machine learning workflows.
