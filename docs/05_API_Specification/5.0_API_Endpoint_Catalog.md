# Purpose

This document provides a high-level catalog of the REST API endpoints exposed by the ExperiML backend.

It serves as an API inventory for frontend developers, backend developers, testers, and project reviewers by listing the available resources, supported operations, and authentication requirements.

Detailed API implementation, request validation, response structures, and architectural standards are documented in **Chapter 4 – Backend Design**.

---

# API Overview

ExperiML exposes a versioned REST API over HTTPS.

**Base URL**

```text
/api/v1
```

General API characteristics:

- RESTful architecture
- JSON request and response bodies
- JWT-based authentication for protected endpoints
- Standardized API responses
- Versioned endpoint structure
- Consistent HTTP status codes
- Pagination support for collection endpoints

# Authentication

Protected endpoints require a valid JWT access token.

The token must be included in the HTTP Authorization header using the Bearer authentication scheme.

```text
Authorization: Bearer <jwt-token>
```

Authentication and authorization mechanisms are described in **Chapter 4.5 – Authentication & Security**.

---

# Standard API Responses

All endpoints follow a standardized response structure.

Successful responses:

```json
{
  "success": true,
  "message": "Operation completed successfully.",
  "data": {}
}
```

Error responses:

```json
{
  "success": false,
  "message": "Operation failed.",
  "errors": []
}
```

Detailed response standards are documented in **Chapter 4.4 – API Design & Standards**.

# Pagination

Collection endpoints support pagination through query parameters.

Example:

```text
GET /api/v1/projects?page=1&limit=10
```

Pagination behavior is standardized across all collection endpoints and is documented in **Chapter 4.4 – API Design & Standards**.

# API Versioning

ExperiML currently exposes Version 1 of its REST API.

```text
/api/v1
```

Future breaking changes will be introduced through new API versions while maintaining backward compatibility where appropriate.

# Design Decisions

The API specification follows these principles:

- Single versioned REST API.
- Resource-oriented endpoint design.
- Consistent endpoint naming conventions.
- JWT authentication for protected resources.
- Standardized response structures.
- Uniform HTTP status codes.
- Consistent pagination strategy.
- Separation between endpoint catalog and implementation details.

---

# Summary

The ExperiML API provides a consistent, versioned REST interface for managing users, projects, datasets, experiments, artifacts, and predictions.

This catalog serves as a concise reference for available endpoints while deferring implementation details, validation rules, and architectural standards to the Backend Design documentation.
