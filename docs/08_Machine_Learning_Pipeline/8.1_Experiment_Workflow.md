# Purpose

This document describes the complete lifecycle of a machine learning experiment within ExperiML.

It explains how users interact with the platform to create, configure, execute, monitor, and review machine learning experiments while maintaining reproducibility and organized experiment management.

---

# Overview

Experiments are the central entity within ExperiML.

Each experiment represents a complete machine learning workflow consisting of a dataset, selected algorithm, configuration parameters, training execution, evaluation results, generated artifacts, and prediction outputs.

The platform ensures that every experiment follows a standardized workflow, allowing users to reproduce results, compare different models, and maintain an organized history of their machine learning work.

# Workflow Stages

## Project Creation

Every experiment belongs to a project.

Projects provide logical organization by grouping related datasets, experiments, artifacts, and prediction results.

---

## Dataset Selection

Users either upload a new dataset or select an existing dataset associated with the project.

Only validated datasets are available for experimentation.

---

## Experiment Creation

A new experiment is created by providing basic experiment information, including:

- Experiment name
- Associated dataset
- Machine learning task
- Description (optional)

Each experiment receives a unique identity for tracking and management.

---

## Experiment Configuration

Users configure the experiment by selecting:

- Machine learning algorithm
- Target variable
- Feature selection
- Hyperparameters
- Training configuration

These settings define how the model will be trained and evaluated.

---

## Training Execution

After configuration, the experiment enters the training phase.

The Express backend coordinates the workflow while the FastAPI service performs the computational tasks.

Training status is tracked throughout execution.

---

## Progress Monitoring

Users can monitor the status of an experiment during execution.

Typical experiment states include:

- Pending
- Running
- Completed
- Failed

This allows users to track long-running training jobs without blocking other platform activities.

---

## Result Review

After training completes, the experiment dashboard provides access to:

- Evaluation metrics
- Model performance summaries
- Generated visualizations
- Training configuration
- Execution information

These results allow users to analyze and compare model performance.

---

## Artifact Management

Every completed experiment generates reusable artifacts.

Examples include:

- Trained model
- Evaluation metrics
- Visualization files
- Configuration metadata
- Prediction outputs

Artifacts remain linked to the originating experiment for future reference.

---

## Prediction Generation

Users may use trained models to generate predictions using new datasets.

Prediction outputs become part of the experiment history and can be downloaded or reviewed later.

---

# Experiment Lifecycle States

Experiments transition through a defined set of lifecycle states.

| State | Description |
|---------|-------------|
| Draft | Experiment has been created but not started |
| Pending | Waiting for execution |
| Running | Training is currently in progress |
| Completed | Training finished successfully |
| Failed | Training terminated due to an error |
| Archived | Experiment retained for historical reference |

These states help users monitor experiment progress and maintain organized project histories.

---

# Workflow Characteristics

The experiment workflow follows several design principles.

- Standardized execution process
- Reproducible experiment configuration
- Persistent experiment history
- Independent training execution
- Centralized metadata management
- Reusable experiment artifacts
- Clear lifecycle tracking

These characteristics provide a consistent and maintainable experimentation experience.

---

# Design Decisions

The workflow is based on the following principles:

- Every experiment belongs to a project.
- Every experiment uses one validated dataset.
- Experiment configurations remain immutable after execution to preserve reproducibility.
- Training is executed asynchronously through the FastAPI service.
- Results and artifacts remain permanently associated with the originating experiment.
- Lifecycle states provide clear execution tracking.

# Summary

ExperiML provides a structured experiment workflow that guides users from project creation through dataset selection, experiment configuration, model training, result analysis, artifact management, and prediction generation.

By enforcing a standardized lifecycle and maintaining complete experiment histories, the platform enables organized, reproducible, and scalable machine learning experimentation.
