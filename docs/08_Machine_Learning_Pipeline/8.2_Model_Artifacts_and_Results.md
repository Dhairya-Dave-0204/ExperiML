# Purpose

This document describes the outputs generated by machine learning experiments within ExperiML.

It defines the different categories of artifacts, evaluation results, prediction outputs, and metadata produced during the machine learning workflow, as well as their storage and retrieval strategy.

---

# Overview

Every completed experiment generates a collection of outputs that collectively represent the results of the training process.

These outputs allow users to evaluate model performance, reproduce experiments, generate predictions, and compare different machine learning approaches.

All generated artifacts remain permanently associated with their originating experiment unless explicitly removed by the user.

---

The generated outputs become part of the experiment history and remain available for future analysis.

# Artifact Categories

ExperiML organizes generated outputs into several categories.

| Category | Purpose |
|----------|---------|
| Trained Model | Serialized machine learning model |
| Evaluation Metrics | Performance measurements |
| Visualizations | Charts and evaluation plots |
| Prediction Outputs | Generated prediction files |
| Experiment Metadata | Configuration and execution information |

This organization simplifies retrieval and long-term experiment management.

---

# Trained Models

Each completed experiment generates a serialized model that can be reused for future prediction tasks.

The trained model represents the final outcome of the learning process and serves as the primary artifact for deployment or inference.

Examples include:

- Classification models
- Regression models
- Clustering models

The model file is securely stored and linked to the corresponding experiment.

---

# Evaluation Results

ExperiML generates evaluation results to measure model performance.

Depending on the selected machine learning task, evaluation may include:

### Classification

- Accuracy
- Precision
- Recall
- F1 Score
- ROC-AUC

### Regression

- MAE
- MSE
- RMSE
- R² Score

The generated metrics provide quantitative insight into model quality and enable comparison across experiments.

---

# Visualization Outputs

Visualizations improve interpretability by presenting model performance graphically.

Generated visualizations may include:

- Confusion Matrix
- ROC Curve
- Precision-Recall Curve
- Feature Importance
- Residual Plots
- Learning Curves

Visualizations are stored as experiment artifacts and can be viewed or downloaded.

---

# Prediction Outputs

After successful training, users can generate predictions using new datasets.

Prediction outputs typically include:

- Predicted values
- Prediction probabilities (where applicable)
- Generated output files
- Prediction metadata

Prediction results remain associated with the experiment that produced them.

---

# Experiment Metadata

Each experiment maintains metadata that supports reproducibility and traceability.

Metadata includes:

- Experiment identifier
- Dataset reference
- Selected algorithm
- Hyperparameters
- Training timestamp
- Execution status
- Artifact references

Metadata is stored in the database, while generated files are stored separately within the file storage system.

---

# Storage Strategy

ExperiML separates generated files from relational data.

| Component | Responsibility |
|-----------|----------------|
| Database | Metadata |
| File Storage | Models, charts, prediction files, reports |

This separation improves scalability while simplifying storage management and backup strategies.

---

# Result Retrieval

Users can access experiment outputs through the application interface.

Available operations include:

- View experiment summary
- View evaluation metrics
- Preview visualizations
- Download trained model
- Download prediction results
- Download generated artifacts

All retrieval operations require authentication and resource ownership verification.

---

# Pipeline Characteristics

The artifact management process follows several architectural principles.

- Persistent experiment history
- Reproducible outputs
- Centralized metadata
- Organized artifact storage
- Secure access control
- Reusable trained models
- Consistent result management

These principles ensure that generated outputs remain organized, traceable, and accessible throughout the lifecycle of an experiment.

---

# Design Decisions

The artifact management architecture follows these principles:

- Every completed experiment generates reusable artifacts.
- Generated files remain associated with their originating experiment.
- Metadata is stored separately from binary files.
- Evaluation results are preserved for experiment comparison.
- Prediction outputs are treated as first-class experiment artifacts.
- Secure access controls govern artifact retrieval.

---

# Summary

ExperiML provides a structured artifact management system that preserves every significant output generated during machine learning experiments.

By separating metadata from generated files, organizing artifacts by experiment, and providing secure retrieval mechanisms, the platform ensures that experiment results remain reproducible, accessible, and easy to manage throughout their lifecycle.

---

# Architecture Decision Record

**ADR-102:** ExperiML stores all experiment outputs—including trained models, evaluation metrics, visualizations, prediction results, and configuration metadata—as persistent artifacts associated with their originating experiment. Metadata is maintained within the relational database, while generated files are stored separately to improve scalability, maintainability, and reproducibility.
