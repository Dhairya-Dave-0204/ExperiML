# Purpose

This document provides a high-level overview of the Machine Learning (ML) pipeline implemented within ExperiML.

The pipeline defines how datasets progress through validation, preprocessing, model training, evaluation, artifact generation, and prediction. It establishes a standardized workflow that enables reproducible machine learning experiments while maintaining consistency across different algorithms and datasets.

---

# Overview

ExperiML follows a structured machine learning workflow that separates experiment management from model execution.

The Node.js backend is responsible for project management, experiment orchestration, authentication, and metadata management, while the FastAPI service performs computationally intensive machine learning tasks.

This separation improves modularity, scalability, and maintainability while allowing the ML engine to evolve independently of the application backend.

---

# ML Pipeline Architecture

The overall machine learning workflow is illustrated below.

```text
                     Dataset Upload
                           │
                           ▼
                  Dataset Validation
                           │
                           ▼
                   Data Preprocessing
                           │
                           ▼
                  Experiment Creation
                           │
                           ▼
                 Training Configuration
                           │
                           ▼
                     Model Training
                           │
                           ▼
                    Model Evaluation
                           │
                           ▼
                  Artifact Generation
                           │
                           ▼
                  Prediction Generation
                           │
                           ▼
              Metadata & Result Storage
```

Each stage represents a logical step within the machine learning lifecycle and is executed in a controlled sequence.

---

# Pipeline Stages

## Dataset Upload

The workflow begins when a user uploads a supported dataset through the application interface.

Uploaded datasets are validated and securely stored before becoming available for experimentation.

---

## Dataset Validation

The uploaded dataset undergoes validation to ensure compatibility with the platform.

Validation may include:

- Supported file format verification
- Dataset integrity checks
- Structural validation
- Metadata generation

Only valid datasets proceed to the next stage.

---

## Data Preprocessing

Before training begins, the dataset is prepared for machine learning.

Typical preprocessing activities include:

- Missing value handling
- Feature encoding
- Feature scaling (where applicable)
- Dataset splitting
- Feature selection

The preprocessing strategy depends on the selected machine learning algorithm and experiment configuration.

---

## Experiment Configuration

Users configure an experiment by selecting:

- Dataset
- Machine learning algorithm
- Target variable
- Hyperparameters
- Evaluation strategy

The experiment configuration defines how the training process will be executed.

---

## Model Training

The FastAPI service performs model training using the configured dataset and experiment settings.

Training is executed independently of the main application server to prevent long-running computational tasks from affecting overall system responsiveness.

---

## Model Evaluation

After training completes, the generated model is evaluated using appropriate performance metrics.

Depending on the learning task, evaluation may include:

- Classification metrics
- Regression metrics
- Cross-validation results
- Model performance summaries

Evaluation results become part of the experiment record.

---

## Artifact Generation

The training process produces several reusable artifacts.

Examples include:

- Trained model
- Evaluation metrics
- Prediction outputs
- Visualization files
- Configuration metadata

Artifacts are securely stored and associated with their corresponding experiment.

---

## Prediction Generation

Trained models may be used to generate predictions using new input data.

Prediction results are stored as experiment outputs and remain available for later review or download.

---

# System Responsibilities

The machine learning workflow is distributed across multiple system components.

| Component | Responsibility |
|-----------|----------------|
| Frontend | User interaction and experiment management |
| Express Backend | Authentication, orchestration, metadata management |
| FastAPI Service | Machine learning execution |
| Database | Metadata storage |
| File Storage | Datasets, models, artifacts, prediction outputs |

This separation improves scalability and simplifies maintenance.

---

# Pipeline Characteristics

The ExperiML pipeline follows several architectural principles.

- Modular execution
- Reproducible experiments
- Consistent preprocessing workflow
- Independent model execution
- Centralized metadata management
- Artifact persistence
- Scalable service separation

These principles ensure that experiments remain organized, repeatable, and maintainable.

---

# Design Decisions

The machine learning architecture is based on the following decisions:

- Experiment orchestration is handled by the Express backend.
- Computational workloads are delegated to the FastAPI service.
- Datasets and generated artifacts are stored outside the relational database.
- Metadata is centrally managed through the database.
- Pipeline stages execute sequentially to maintain consistency and reproducibility.

---

# Summary

ExperiML implements a structured machine learning pipeline that standardizes the lifecycle of every experiment, from dataset upload through model training and evaluation to artifact generation and prediction.

By separating orchestration from computation and maintaining centralized metadata alongside persistent artifacts, the platform provides a scalable, maintainable, and reproducible environment for machine learning experimentation.
